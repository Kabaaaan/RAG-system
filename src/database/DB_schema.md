# Схема базы данных 

### 1. `resource_types` — Справочник типов ресурсов

| Поле     | Тип         | Ограничения          | Описание                                      |
|----------|-------------|----------------------|-----------------------------------------------|
| `id`     | `int`       | PK, AUTO_INCREMENT   | Уникальный идентификатор типа ресурса         |
| `name`   | `varchar`   | NOT NULL, UNIQUE     | Название типа (например: `article`, `faq`, `product`, `news`) |

**Назначение:** Справочник для нормализации типов ресурсов.

---

### 2. `rag_resources` — Основное хранилище ресурсов (DataLake / staging-area)

| Поле           | Тип         | Ограничения                  | Описание                                      |
|----------------|-------------|------------------------------|-----------------------------------------------|
| `id`           | `bigint`    | PK, AUTO_INCREMENT           | Уникальный ID ресурса (используется везде)   |
| `url`          | `varchar`   | NULLABLE                     | URL оригинального ресурса (если есть)         |
| `type_id`      | `int`       | NOT NULL, FK → `resource_types.id` | Тип ресурса                               |
| `title`        | `varchar`   | NULLABLE                     | Заголовок ресурса                             |
| `text`         | `text`      | NOT NULL                     | Основное содержимое ресурса (для векторизации)|
| `created_at`   | `timestamp` | DEFAULT CURRENT_TIMESTAMP    | Дата и время создания записи                  |

**Важные особенности:**
- Ресурсы **только добавляются**. Обновление и удаление **не поддерживаются**.
- После вставки автоматически запускается задача на индексацию в векторную БД.
- `text` — это поле, которое в первую очередь отправляется в эмбеддинг-модель.

---

### 3. `recommendation_types` — Справочник типов рекомендаций

| Поле     | Тип         | Ограничения          | Описание                                      |
|----------|-------------|----------------------|-----------------------------------------------|
| `id`     | `bigint`    | PK, AUTO_INCREMENT   | Уникальный идентификатор типа рекомендации    |
| `name`   | `varchar`   | NOT NULL, UNIQUE     | Название типа (например: `similar_content`, `next_step`, `cross_sell`) |

**Назначение:** Справочник для расширяемых типов рекомендаций.

---

### 4. `recommendations` — Хранилище сгенерированных рекомендаций

| Поле           | Тип         | Ограничения                  | Описание                                      |
|----------------|-------------|------------------------------|-----------------------------------------------|
| `id`           | `bigint`    | PK, AUTO_INCREMENT           | Уникальный ID рекомендации                    |
| `lead_id`      | `bigint`    | NOT NULL                     | Идентификатор пользователя (lead)             |
| `type_id`      | `int`       | NOT NULL, FK → `recommendation_types.id` | Тип рекомендации                     |
| `text`         | `text`      | NOT NULL                     | Содержимое рекомендации (JSON или текст)     |
| `created_at`   | `timestamp` | DEFAULT CURRENT_TIMESTAMP    | Дата и время генерации рекомендации           |

**Важные особенности:**
- Одна запись = одна рекомендация определённого типа для конкретного `lead_id`.
- При генерации рекомендации (`POST /recommendations/generate`) данные сохраняются именно в эту таблицу.

---

**Индексы**  
   - `rag_resources` — индекс по `type_id`  
   - `recommendations` — индекс по `lead_id`
