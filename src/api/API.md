# API Documentation

## Общая информация

API предназначено для работы с системой рекомендаций и векторной базой данных.  
Все длительные операции (генерация рекомендаций, обновление векторной БД) выполняются асинхронно через очередь сообщений.

### Авторизация
Способ не выбран, скорее всего JWT. 

### Формат данных
- **Content-Type:** application/json
- **Кодировка:** UTF-8

---

## 1. Генерация рекомендации 
### `POST /recommendations/generate`

Создает задачу на генерацию рекомендации для пользователя.


### Request Body
```json
{
  "lead_id": "string",
  "type": "string"
}
````

### Params

| Поле    | Тип    | Обязательное | Описание                       |
| ------- | ------ | ------------ | ------------------------------ |
| lead_id | string | да           | Идентификатор пользователя     |
| type    | string | да           | Тип рекомендации (расширяемый) |

### Response

```json
{
  "token": "string",
  "status": "queued"
}
```

### Описание статусов

* `queued` — задача поставлена в очередь

---

## 2. Получение статуса рекомендации
### `GET /recommendations/status`

Позволяет отслеживать состояние генерации рекомендации.

### Query Params

| Параметр | Тип    | Обязательный | Описание           |
| -------- | ------ | ------------ | ------------------ |
| token    | string | да           | Токен рекомендации |

### Response

```json
{
  "status": "string"
}
```

### Возможные статусы

* `queued` — в очереди
* `processing` — генерируется
* `completed` — сохранена в БД
* `failed` — ошибка генерации

---

## 3. Обновление ресурса в векторной БД
### `POST /vector-db/update`

Создает задачу на обновление конкретного ресурса в векторной БД.

### Request Body

```json
{
  "resource_type": "string",
  "resource_id": "string"
}
```

### Params

| Поле          | Тип    | Обязательное | Описание                 |
| ------------- | ------ | ------------ | ------------------------ |
| resource_type | string | да           | Тип ресурса              |
| resource_id   | string | да           | ID ресурса в основной БД |

### Response

```json
{
  "operation_id": "string",
  "status": "queued"
}
```

### Описание

Операция асинхронная, выполняется через очередь сообщений.

---

## 4. Полное обновление векторной БД
### `POST /vector-db/rebuild`

Запускает полную переиндексацию векторной базы данных.

### Request Body

```json
{}
```

### Response

```json
{
  "operation_id": "string",
  "status": "queued"
}
```

---

## 5. Статус векторной БД
### `GET /vector-db/status`

Позволяет узнать текущее состояние векторной базы.

### Response

```json
{
  "status": "string"
}
```

### Возможные статусы

* `updating` — идет обновление
* `ready` — актуальна и доступна

---

## 6. Получение рекомендации пользователя
### `GET /recommendations`

Возвращает сохраненную рекомендацию для пользователя.

### Query Params

| Параметр | Тип    | Обязательный | Описание                   |
| -------- | ------ | ------------ | -------------------------- |
| lead_id  | string | да           | Идентификатор пользователя |

### Response

```json
{
  "lead_id": "string",
  "recommendations": [
    {
      "id": "string",
      "type": "string",
      "data": {}
    }
  ]
}
```

---

## 7. Статус ресурса в векторной БД
### `GET /vector-db/resource-status`

Позволяет узнать, индексирован ли ресурс.

### Query Params

| Параметр      | Тип    | Обязательный | Описание    |
| ------------- | ------ | ------------ | ----------- |
| resource_id   | string | да           | ID ресурса  |
| resource_type | string | да           | Тип ресурса |

### Response

```json
{
  "status": "string"
}
```

### Возможные статусы

* `not_found` — отсутствует в векторной БД
* `processing` — в процессе создания
* `created` — успешно добавлен
